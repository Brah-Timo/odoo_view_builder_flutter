A **professional visual drag-and-drop editor** for generating Odoo XML views —
Form, Tree, and Kanban — built with Flutter and running on every platform.


---


## ✨ Features

| Category | What you get |
|----------|-------------|
| 🎨 **Visual Editor** | Drag fields from a typed palette onto a live canvas; organise into groups and notebook pages |
| 🌳 **All View Types** | Form · Tree (List) · Kanban — each with a dedicated editor |
| 👁 **Live XML Preview** | Syntax-highlighted XML updates in real time as you build |
| ✅ **Validation** | Field name rules, XML structure checks, Odoo 14–18 compatibility warnings |
| 📤 **Multi-Platform Export** | Download file · Copy to clipboard · Native share sheet · Web blob download |
| 📥 **Import XML** | Parse existing Odoo XML back into the visual editor |
| 📚 **Template Library** | 10 built-in templates for common models (partner, sale order, product …) |
| 🔗 **Odoo API Connect** | Fetch live field definitions directly from a running Odoo instance |
| ↩ **Undo / Redo** | 50-step history across all edit actions |
| 💾 **Local Persistence** | Views auto-saved to SQLite; settings in SharedPreferences |
| 📊 **Analytics** | Firebase-ready analytics service (works without Firebase too) |
| 🐛 **Crash Reporting** | Crashlytics-ready crash reporter with local log fallback |

## 🚀 Quick Start

### Prerequisites

- Flutter SDK ≥ 3.10.0
- Dart SDK ≥ 3.0.0

### Install

```bash
git clone https://github.com/Brah-Timo/odoo_view_builder_flutter.git
cd odoo_view_builder_flutter
flutter pub get
```

### Run

```bash
# Android
flutter run -d android

# iOS
flutter run -d ios

# Web
flutter run -d chrome

# Desktop
flutter run -d macos      # or windows / linux
```

For detailed platform-specific setup (entitlements, permissions, Firebase)
see **[doc/setup.md](doc/setup.md)**.

## 📦 Output Format

Every exported view is a standard Odoo module XML file that you can drop
directly into your `module/views/` folder:

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- Generated by Odoo View Builder -->
<odoo>
    <data>
        <record id="view_partner_form_custom" model="ir.ui.view">
            <field name="name">res.partner.form.custom</field>
            <field name="model">res.partner</field>
            <field name="arch" type="xml">
                <form>
                    <group string="General Info">
                        <field name="name" required="1"/>
                        <field name="email" widget="email"/>
                        <field name="phone" widget="phone"/>
                    </group>
                    <group string="Address">
                        <field name="street"/>
                        <field name="city"/>
                        <field name="zip"/>
                    </group>
                </form>
            </field>
        </record>
    </data>
</odoo>
```

## 🏗 Architecture

The app follows **Clean Architecture** with three layers:

```
Presentation  →  Domain  →  Data
(Riverpod)       (UseCases)   (sqflite / SharedPrefs / OdooAPI)
```

State management uses **Riverpod 2** (`StateNotifierProvider`) throughout.
XML generation is handled by the `xml` package's `XmlBuilder` API —
never string concatenation.

→ Full architecture details: **[doc/architecture.md](doc/architecture.md)**

## 🧩 Supported Odoo Versions

| Odoo | Form | Tree | Kanban | Notes |
|------|------|------|--------|-------|
| 14.0 | ✅ | ✅ | ✅ | `<sheet>` recommended |
| 15.0 | ✅ | ✅ | ✅ | |
| 16.0 | ✅ | ✅ | ✅ | |
| 17.0 | ✅ | ✅ | ✅ | `attrs`/`states` removed |
| 18.0 | ✅ | ✅ | ✅ | `<list>` preferred |

The compatibility validator flags issues when you switch the target version in
**Settings → Odoo Target Version**.
